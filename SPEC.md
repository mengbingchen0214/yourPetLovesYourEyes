# 桌面宠物 · 护眼伴侣 — 产品规格书 v0.1

## 1. Concept & Vision

**产品名称**：护眼Pet / EyePet

**一句话描述**：复刻 QQ 宠物的情怀，用一只桌面宠物提醒你每 20 分钟休息眼睛。

**核心体验**：
- 一只可爱的宠物长期在你的 Mac 桌面上"生活"
- 安静时呆萌，每 20 分钟主动找你互动，提醒你望远护眼
- 支持上传自家宠物照片，AI 帮你变成 Q 版可爱版本
- 护眼提醒就像宠物自发的关心，而不是闹钟的机械通知

**用户情绪**：温暖、被治愈、"这只好可爱，我要每天看到它"

---

## 2. 核心功能

### 2.1 护眼提醒系统
- 每 20 分钟触发一次休息提醒
- 提醒内容："主人～该让眼睛休息一下啦！望向 20 米外 20 秒～ 👀"
- 可配置间隔（15/20/30/45/60 分钟）
- 可静音/暂停
- 提醒时宠物有特殊动画表现

### 2.2 桌面宠物（核心交互）
- 宠物在桌面上浮动（无边框窗口，始终置顶）
- 可拖动位置，位置自动保存
- 双击宠物打开设置面板
- 宠物有 idle 动画（眨眼/摇晃/发呆）
- 宠物在提醒时执行特殊动作

### 2.3 宠物外观系统
- 默认提供 4 种预设 Q 版动物（猫/狗/兔/熊）
- 用户可自定义上传宠物照片
- 上传后自动处理为 Q 版可爱风格（见 2.4）

### 2.4 宠物图像处理流程
**Step 1**: 用户上传照片（.jpg/.png）
**Step 2**: 自动检测并移除背景（rembg）
**Step 3**: AI Q 版化处理（风格迁移/chibi transformation）
**Step 4**: 预览 + 确认下载
**Step 5**: 应用到桌面宠物

**Q 版化风格**：圆润脑袋、大眼睛、短圆身体、简化四肢（参考 LINE 贴图风格）

### 2.5 设置面板
- 提醒间隔配置
- 提醒音量
- 宠物选择/上传
- 启动项设置（Mac 登录后自启）
- 关于/卸载

## 3. 技术方案

### 3.1 桌面客户端（Mac）
**框架**：Electron + 单 HTML 文件（与 Web 版共用代码）

**为什么用 Electron**：
- npm registry 慢的问题可以通过设置国内镜像解决（.npmrc）
- macOS 原生支持好（通知、系统托盘）
- 窗口管理成熟（无边框置顶窗口）
- 可以打包成 .app 分发

**替代方案（如果 Electron 有问题）**：
- Python + rumps（菜单栏 App，轻量但交互有限）
- Flutter（跨平台，但包体积大）

### 3.2 Web 端（宠物图片处理 + 展示）
**框架**：纯 HTML + Python Flask/FastAPI 后端

**为什么分开**：
- 图片处理（rembg + 风格迁移）必须要有服务器端，不能纯前端
- 桌面客户端体积大，但可以无网运行
- Web 端用来做"演示 + 图片处理服务"，也是小红书引流的落地页

### 3.3 图片处理流程
```
用户上传照片
    ↓
Flask API 接收图片
    ↓
rembg 去背景 → 透明底 PNG
    ↓
风格迁移模型 → Q 版化
    ↓
返回处理后图片 + 下载链接
```

**技术栈**：
- 后端：Python + FastAPI
- 去背景：rembg (RMBG-1.4)
- Q 版化：ControlNet (tile 模型) 或 AnimeGAN
- 模型托管：本地运行 or 付费 API（如 Replicate）

### 3.4 提醒系统
- 纯 JavaScript setInterval（网页端）
- Electron 内置系统通知（桌面端）
- macOS 原生 Notification API

## 4. 小红书商业化路径

### 4.1 变现产品选择

| 方案 | 产品形态 | 价格 | 优势 | 劣势 |
|---|---|---|---|---|
| A | 桌面宠物客户端（免费）+ 高级 Q 版处理（付费） | 9.9 元/20 张 Q 版图 | 轻资产，快速启动 | 用户付费意愿不确定 |
| B | 桌面宠物客户端付费 Pro 版 | 29 元一次性 | 利润高 | 付费门槛高，需要很强差异化 |
| C | 护眼 Pet 桌面端（免费）+ 小红书引流 → 引到微信卖定制宠物图 | 免费 + 按需收费 | 转化链路短 | 需要持续运营 |

**推荐方案 C**：免费桌面端获取用户 → 小红书内容（宠物+Q版图对比）→ 引流到微信私域 → 付费定制 Q 版宠物图（19.9 元/张）

- 教程："如何把自家宠物变成超可爱的 Q 版桌面宠物"

## 8. 开放问题（待确认）

1. **Q 版化是自己训练模型还是用现有模型？**
   - 自训练：效果好但需要数据
   - 现有：AnimeGANv2 / ControlNet tile（快速可用）

2. **是否需要支持 Windows？**
   - 当前规划仅 Mac，后续可扩展

3. **桌面宠物是始终置顶还是可最小化？**
   - 设计为始终置顶 + 可拖动到屏幕边缘自动隐藏

4. **是否需要宠物和用户的对话互动（不止提醒）？**
   - 暂定：纯护眼提醒，不做对话（避免变成玩具而非工具）
